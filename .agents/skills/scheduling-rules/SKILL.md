## 一、班別定義

| 班別 | 代碼 | 時段 | 說明 |
|------|------|------|------|
| 白班 | `day` | ~08:00–16:00 | 一般日班 |
| 白班 8-8 | `day88` | 08:00–20:00 | 12hr 長白班，J 為主力，他人可支援 |
| 小夜班 | `evening` | ~16:00–00:00 | |
| 大夜班 | `night` | ~00:00–08:00 | |
| 大夜 8-8 | `night88` | 20:00–08:00 | 12hr 長大夜，有 day88 時必須搭配，不可連續兩天 |
| 休假 | `off` | — | |

## 二、循環與月度關係

- **循環模板**：4 週 (28 天) 為一個循環，起算日為 **2026/3/15**
- **循環是「規則」**，月度班表是「結果」
- 一個月可能橫跨兩個循環，系統會自動拆分拼接
- 循環編號：C1 = 3/15~4/11, C2 = 4/12~5/9, C3 = 5/10~6/6 ...

## 五、假期規則

### 假期配額計算 (自動)
```
總假日 = (可用人數 × 循環天數) - Σ(每日各班別最低需求)
每人配額 = 總假日 ÷ 可用人數
```
以目前設定：9人 × 28天 = 252，需求 ≈ 188，**總假 ≈ 64 天，每人 ≈ 7 天**

### 預假機制
- 預假由管理者在排班前手動指定日期
- 自動排班時**最高優先**尊重預假
- 預假天數計入該人的總假日配額

### 假期跨月追蹤
- 一個月可能橫跨兩個循環
- 系統需追蹤每人在每個循環中已用 / 剩餘的假日數

## 六、硬性約束 (Hard Constraints — 不可違反)

| 編號 | 規則 | 判斷位置 |
|------|------|----------|
| **R1** | 大夜 → 隔天白班 ❌ | `canAssign()` |
| **R2** | 小夜 → 隔天白班 ❌ | `canAssign()` |
| **R3** | 每人每天最多一班 | `canAssign()` |
| **R4** | 個人禁排班別 (參照員工表) | `canAssign()` |
| **R5** | 個人班別天數上限 (參照員工表) | `canAssign()` |
| **R6** | 至少每 **6 天**休 1 天 | `canAssign()` 雙向檢查 |
| **R7** | 大夜/小夜必須**連續至少 2 天** | `fillConsecutive()` 原子分配 |
| **R8** | **大夜8-8**：必須有白班8-8搭配，且不可連兩日 | 單獨步驟升級處理 |
| **R9** | 各班別達到最低人數 | 補位階段

### R6 判斷邏輯 (重要！)

```go
// 必須「雙向」檢查：往前看 + 往後看
// 計算：當前日期前面連續工作幾天 + 後面已排定連續工作幾天
// 如果 前方連班 + 1 (今天) + 後方連班 > 6 → 禁止排入
// 同時，每排完一段連續班後，必須在其後方標記 "off" 作為強制休假佔位
```

### R7 判斷邏輯 (重要！)

```go
// 大夜/小夜用 fillConsecutive() 分配，一次分配一整段 (如連續 4 天)
// 分配前做「原子性」預檢：整段都能排才排，否則整段跳過
// 補位階段如需新增夜班，優先找「昨天也是這個班的人」延續既有班段
```

## 七、軟性約束 (Soft Constraints — 盡量滿足)

| 編號 | 規則 |
|------|------|
| **S1** | 白班：期望 5 天休 1 天 |
| **S2** | 小夜：期望 4 天休 1 天 |
| **S3** | 大夜：期望 4 天休 2 天 |
| **S4** | 大小夜連續天數最多 6 天 |
| **S5** | 大小夜班次平均分配 |
| **S6** | 大小夜班最好要排就排到連續4天 |

## 九、自動排班演算法步驟 (v6)

```
步驟 0：計算假期配額
  → 總假日 = 可用人次 - 工作需求
  → 每人配額 = 總假日 ÷ 可用人數

步驟 1：鎖定預假 ⭐ (最高優先)
  → 讀取每人的預假設定
  → 在 schedule 中標記為 "off"
  → 扣減該人的假期配額

步驟 2：智慧假期分配 ⭐ (第二優先)
  → 按 R6 (做 6 休 1) 規則自動插入強制休假
  → 優先在人力需求較低的日子安排休假
  → 盡量讓每人的假日均勻散佈在 28 天中

步驟 3：排 J (8-8 主力)
  → J 也適用 R6（做 6 休 1），休假時直接就是 off

步驟 4：排大夜 (A, B, D, I)
  → 連續 4 天 + 休 2 天 模式輪排
  → I 受 ≤ 5 天上限限制
  → 原子性分配，分配後標記後方休假

步驟 4.5：升級大夜為 night88 (R8 處理) ⭐
  → 逐日檢查，若當日有 day88，則需一位 night88
  → 優先尋找已排定 night 且昨日非 night88 的員工升級
  → 若當日僅有的一位大夜昨日已是 night88，則退回以補位方式增派一人支援

步驟 5：排小夜 (C, E, F)
  → 連續 4 天 + 休 1 天 模式輪排
  → 最吃緊班別，需密切檢查可行性

步驟 6：補位白班
  → 填滿各天白班缺口
  → 遵守各人白班天數上限
  → H 僅排白班

步驟 7：驗證所有約束
  → 硬性 R1~R8 全部檢查
  → 統計軟性約束達成率
```
